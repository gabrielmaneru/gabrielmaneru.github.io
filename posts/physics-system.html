Overview:

The goal of this course is to implement a fully functional collision-detection and collision-resolution physics system. Besides, extra features can be implemented such as destructible objects, stacking, soft bodies or friction.

Implementation details:

Half-Edge Mesh:

There are many popular data structures used to store polygonal meshes. The one I use in this project is called half-edge data structure. Instead of storing their connectivity in the faces referring to the neighbor vertices, this structure is edge-based and so will store the connectivity information on the edges. Each edge will reference the vertex it is facing, the face it belongs, the next edge and the twin edge being the opposite edge. Having this construction allows circulating the face accessing directly neighboring faces. This allows, for example, to easily retrieve the support vertex along a given direction just by hill-climbing through the cycles of edges.

Gilbert-Johnson-Keerthi distance algorithm:

The GJK algorithm is a method for determining the distance between two shapes. Its peculiarity is that instead of needing the full comprehension of how the geometry is stored, it only relies on a support function (returning the point on the shape which has the largest elongation in a given direction). The objective if to iteratively generate the closer simplex to the origin of the Minkowski difference.

Showcase: (Black: Minkowski difference | White: Simplex Closer to the origin)

If we couldn't find a simplex in the Minkowski difference that englobes the origin, then there is no intersection, job done. However if the algorithm manages to find a simplex that does so, then we should retrieve the contact information to resolve the collision. We need to find the faces of the shapes that create the closer face to the origin, inside the Minkowski space.

Expanded Polytope algorithms:

The EPA algorithm uses the simplex from the GJK algorithm and expands it into a polytope in the direction of the normal of the face closer to the origin. We continue the iterative expansion until we find no further vertices along that expansion direction. We consider that last face to be the contact face. We can then extract the contact information from that face.

Showcase: (Blue: GJK Simplex | Green: EPA Polytope | Red: Contact Face)

From the output of EPA, we could retrieve the faces of contact, as well as the barycentric coordinates to compute the two intersection points, the normal and the penetration.

Showcase: (Red/Green: Vertex of the faces of contact | Black: Intersection points | Yellow: Intersection normal and penetration)
